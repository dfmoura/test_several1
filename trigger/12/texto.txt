responta:
Não — não é impeditivo, mas pode ser um limitador se for a única forma de acesso.

E aqui vale separar duas coisas que muita gente mistura:

“ser mobile” não é problema
“ser apenas mobile” pode ser problema no B2B

1) No seu caso, mobile é até positivo

Para o seu público (condomínios / síndicos), mobile ajuda em:

consulta rápida de saldo e cobranças
envio de boletos / PDFs no WhatsApp
acompanhamento simples do mês
acesso em tempo real

Ou seja:

✔ mobile funciona muito bem para “visualização e operação leve”
2) Onde mobile começa a limitar

Seu sistema tem uma característica importante:

ele é um sistema financeiro completo (quase um ERP)

E isso normalmente envolve:

lançamentos mais complexos
conferência de extratos
ajustes e correções
relatórios detalhados

Essas tarefas:

❌ são mais difíceis de fazer confortavelmente só no celular
3) O risco real não é o app mobile — é a percepção

Se você posiciona como:

“app de condomínio no celular”

o mercado tende a pensar:

simples demais
limitado
amador (mesmo não sendo)

Mas seu sistema não é isso.

4) O jeito certo de enxergar seu produto

Você não construiu “um app mobile”.

Você construiu:

um sistema financeiro de condomínio com interface mobile

Isso muda completamente a percepção.

5) O modelo mais forte (o que empresas fazem)

Os sistemas mais fortes do seu tipo são:

backend + regras financeiras completas
painel web (administração pesada)
mobile (consulta + ações rápidas)
6) Então mobile é um problema? Depende da arquitetura
✔ NÃO é problema se:
o mobile for para uso diário simples
a lógica principal já está resolvida
você usa ele como “terminal de operação”
⚠️ pode ser limitação se:
toda configuração complexa depende só do celular
relatórios e ajustes forem difíceis de usar
o síndico precisar “pensar como usuário de sistema pesado” no mobile
7) O ponto mais importante no seu caso

Você já tem algo raro:

automação financeira real + integração bancária + fluxo completo

Isso não é limitado pelo formato mobile.

O que pode limitar crescimento é só:

UX para operações complexas

8) Estratégia inteligente (sem retrabalho grande)

Você NÃO precisa abandonar mobile.

Você pode evoluir assim:

Fase 1 (agora)

✔ mobile como produto principal
✔ foca em simplicidade + venda

Fase 2 (quando tiver clientes)

✔ adiciona painel web simples (admin / ajustes)

Isso resolve 90% dos problemas de percepção de “ERP sério”.

9) Resumo honesto
❌ não é impeditivo
❌ não te impede de vender
✔ é até vantagem para uso diário
⚠️ pode limitar escala se não houver visão web/admin no futuro
10) Vou te dizer o ponto mais importante de todos

Seu problema hoje não é:

mobile
banco Inter
tecnologia
nem Play Store

É só um:

transformar um sistema técnico completo em um produto fácil de entender e comprar em 10 segundos

Se quiser, posso te ajudar a dar o próximo passo mais estratégico:

definir se você deve ou não adicionar painel web agora
ou montar a sua proposta comercial que “esconde a complexidade” do mobile
ou desenhar o modelo ideal de produto (mobile + backend + vendas)

Só me diz pra onde você quer evoluir agora.
